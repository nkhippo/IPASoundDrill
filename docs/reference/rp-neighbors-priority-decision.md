# 判断レポート — RP用 neighbors 再計算の優先度

> 作成日: 2026-06-26
> 判断者: Claude（データ検証に基づく）
> 対象: Mode B の distractor 生成に使う `neighbors`（現状 GA 基準）
> 結論: **当面 再計算は不要（低優先・保留）。** GA neighbors の RP 流用で実用上問題なし。

---

## 1. 論点

`neighbors` は GA の IPAトークン編集距離で全3,059語に事前計算済み。
RP は非rhotic・BATH分裂・LOT分裂があるため、GAの近傍関係が RP では変わる。
→ **RP選択時に neighbors を作り直す必要があるか？**

Mode B の distractor は「音素近傍2 ＋ 同バンドrandom1」。狙いはMCQをミニマルペア知覚テストにすること。
よって判断基準は **「GAの近傍が RP でも distractor として機能するか」**。

---

## 2. 検証（実データ・3,059語の全neighborsペア）

長音符 `ː` を母音の長さとして正規化したうえで、GAの各近傍ペアが RP のトークン距離でも近傍（≤2）に留まるかを測定。

| 指標 | 結果 |
|------|------|
| 全近傍ペア数 | 17,541 |
| **RPでも近傍維持（dist≤2）** | **95.4%** |
| RPで近傍から外れる | 4.6% |
| GAミニマルペア（dist=1）のうち RPでも tight（≤1） | **93.5%** |
| 〃 RPで緩む | 6.5% |

### 壊れる6.5%の正体（パターンは明確）

すべて **RPで実際に音が分かれる正当な箇所**。誤誘導ではなく「RPでは紛らわしくなくなる」だけ:

| パターン | 例 | GA | RP |
|----------|----|----|----|
| rhotic | around↔found | /ɚaʊnd/ vs /faʊnd/（近い） | /əraʊnd/ vs /faʊnd/（離れる） |
| BATH分裂 | bad↔bath | 両方 /æ/（ミニマルペア） | /bæd/ vs /bɑːθ/（母音が割れる） |
| LOT分裂 | bar↔boss | 両方 /ɑ/ | /bɑː/ vs /bɒs/ |

→ これらは「GAでは紛らわしいが RP では紛らわしくない」ペア。RP学習者には distractor として**弱くなるだけで、MCQを壊さない**。

---

## 3. MCQ成立性（最悪ケース）

壊れた近傍を**全部捨てた**最悪条件でも:

| 語あたり有効近傍数 | 割合 |
|-------------------|------|
| 2語以上 | **90.6%** |
| ちょうど1語 | 8.7% |
| 0語 | 0.7% |

`<2` の語は設計どおり **同バンドrandom補填**で4択が必ず成立。
→ **GA neighbors 流用でも Mode B の RP運用は破綻しない。**

---

## 4. 結論と推奨

### 結論: RP neighbors 再計算は **低優先・保留**

- GAの近傍は RP で 95%超が維持。distractor品質の劣化は「一部のペアが弱くなる」程度で、誤りや破綻はない
- MCQ成立性は最悪ケースでも90.6%が近傍2語確保＋random補填で担保
- Mode B は稼働直後。RP併用の実利用がどれだけ発生するか未知の段階で、3,059語のRP距離再計算＋再マージのコストを払う合理性が低い

### 再計算に着手すべきトリガー（将来）

以下が観測されたら `gen_neighbors.py` を `rp_ipa` ベースで再実行する:

1. RP利用者から「MCQの選択肢が音的に紛らわしくない（簡単すぎる）」という実フィードバック
2. RPを既定アクセントにする方針変更
3. BATH/LOT/rhotic 系の語が Mode B 出題で高頻度になり、distractor弱化が体感される

### 再計算する場合の手順（メモ）

- `gen_neighbors.py` の距離計算入力を `ipa` → `rp_ipa` に切替
- トークナイザに RP多重記号（`əʊ ɪə eə ʊə ɜː ɑː ɔː iː uː`）を追加、`ː` は長さとして母音に内包（独立トークンにしない）
- 出力を `neighbors_rp` フィールドに格納（GA `neighbors` は残す）
- Mode B は `ACCENT==='rp'` のとき `neighbors_rp` を参照
- 語彙追加時は GA/RP 両方を再生成

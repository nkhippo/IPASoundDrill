設計判断まとめ

対話で確定した仕様と、その理由

各画面 UI の背景にある意思決定を集約。実装・引き継ぎ時の判断根拠として参照してください。

マーキング（覚えた）
◆ユーザーが手動でチェック — システムは正誤で自動評価しない。正解に辿り着く道のりが長く、毎回スペル/IPA を丁寧に入力するとは限らないため、習得の主観判断はユーザーに委ねる。
◆最大3回で卒業 — 3つ塗ったらその単語は卒業。チェックの少ない語から優先出題し、弱点に集中できる。
◆目的ごとに独立 — 「音の発音を確かめる」で think にチェックしても、「音から単語を覚える」の think には反映しない。4目的で別管理。
◆Local Storage 管理 — アカウント登録不要（BE 管理を避ける）。端末内に保存。
◆狙い — 5,000語×4モードは学習記録には不向き。多くの語に軽く触れ、サクサク進める体験を優先。
判定・アクセント
◆完全一致判定（AI不介在） — 正解のスペル/IPA と完全一致のみ正解。空欄でも1文字違いでも同じ「不正解」。惜しさ・部分正解は示さない。
◆GA / RP はセッション共通 — アクセントは全問通して固定。収録音声も IPA も異なるため、学習中は切り替え不可。
◆プロフィールで設定 — 学習プロフィール（3a）でアクセントを選択。各画面はヘッダーに固定バッジで表示。
◆語彙リストは両表記 — 辞書（3b）は GA/RP 双方を個別スピーカーで即参照でき、外部サイトへの離脱を防ぐ。
レベル・構造
◆CEFR は単語ごと — レベルは語によって変わる概念。プロフィールで複数レベルを選択し、出題は A1→B1 と混ざる。各カードに「語彙」タグで表示（アクセントとは別概念）。連結する音に慣れる（2d）を含む全4ドリルでタグを表示。ただし 2d にフィルタ UI は追加しない。
◆既定 A1＋A2 — 初級を既定に。難しすぎる語で離脱しないよう、選んだレベルのみ出題。
◆目的ファースト — 12パラメータの一括提示を廃止し、目的4カードで1タップ開始。「詳しい設定」は top から撤去。
◆絞り込みはインライン — セッション内絞り込みは各ドリル画面（2a–2d）内の静かなチップとして実装。独立画面は設けない。
入場は毎セッション — 目的カード → 学習プロフィール（3a・毎セッション通過／LS プリセット）→ ドリル。「そのままでいいか」の1判断のみ。
入場・オンボーディング
◆プロフィール一元通過 — すべての設定は 3a プロフィールに集約。毎セッション必ず 3a を通過し、旧「詳しい設定」12パラメータもここに集約。分散配置はしない。
◆LS プリセット — 前回設定を LocalStorage で記憶し初期値に。判断負荷は「そのままでいいか」の1つのみ。
◆初回オンボーディング — 初回訪問時のみ 4スライドを自動発火。スキップも完了扱い（LS onboarding_completed_v1）。以降は直接トップ（1a）から開始。
◆任意再表示 — ヘッダーのガイドアイコンからいつでも再表示できる。
各ドリルの要点
◆音から単語を書く — 音＋IPA記号から単語を想起しスペル入力。音→単語の想起訓練。
◆発音から書いてみる — 綴りから IPA を GA/RP のキーボードで組み立てる産出タスク。
◆音から単語を覚える — 音声（＋IPAヒント）から単語を思い出す。近い語がない単語も多いため選択肢は出さない。
◆連結する音に慣れる — 文章中に埋め込んだフレーズを確認。リンキングは速く読む中で生じるため文脈提示。音声はフレーズのみ再生（データ制約）。

## 3. 現状の実装マップ

### 3.1 既存ヘルパー（`index.html`）

```text
activeIpa(c)        … 運用アクセントの phonemic（GA: ipa / RP: rp_ipa）
altIpa(c)           … 反対アクセントの phonemic（GA時→rp_ipa、RP時→ipa）
activeNarrowIpa(c)  … 運用アクセントの narrow（ipa_actual_ga / ipa_actual_rp）
hasNarrowDifference … narrow ≠ phonemic のとき true
```

### 3.2 画面別 — 反対アクセントの表示有無

| 画面 | 学習モード | 練習モード | 反対アクセント表示 | 備考 |
|------|-----------|-----------|-------------------|------|
| **Decode 出題中** | 発音再学習 | 一単語 / 連結音 / 弱形 | ❌ なし | `#dIpa` は `activeNarrowIpa` のみ |
| **Encode 出題中** | 発音再学習 | 一単語 | ❌ なし | 単語表示 + IPA キーボード |
| **Reveal（答え合わせ後）** | 発音再学習 | 一単語・連結音・弱形 | ✅ **一部あり** | `#rAltIpa`（下記） |
| **Mode B Study** | 音から語彙 | — | ❌ なし | `#mbSIpa` のみ |
| **Mode B MCQ / Dictation** | 音から語彙 | — | ❌ 出題中なし | 不正解時は Reveal 経由で同上 |
| **Summary 復習リスト** | 共通 | 共通 | ❌ なし | `activeIpa` のみ |
| **語彙ブラウザ Words** | — | — | △ **両方常時** | GA 行 + 異なるときだけ RP 行（同一時は RP 行非表示） |
| **語彙ブラウザ Phrases** | — | — | △ 同上 | connected_speech.json |

### 3.3 Reveal パネルの現行ロジック（`refreshRevealIpa`）

表示要素は最大 3 段:

1. **主 IPA** (`#rIpa`) … `activeNarrowIpa(c)` — 大きく中央、音素タップ可  
2. **辞書表記** (`#rDictIpa`) … narrow ≠ phonemic のとき `辞書表記: {activeIpa}`  
3. **反対アクセント** (`#rAltIpa`) … `altIpa(c) !== activeIpa(c)` のときのみ  
   - 例（GA 運用）: `RP（英）: /ˈæˌdres/`  
   - **文字列が完全一致すると非表示**（同一扱いで何も出さない）

```text
// 現状（簡略）
if (alt && alt !== currentPhonemic) {
  altEl.textContent = label + ": " + alt;
  show;
} else {
  hide;  // ← 同一時はユーザーに伝えない
}
```

### 3.4 データの有無

| データセット | GA `ipa` | RP `rp_ipa` | narrow |
|-------------|----------|-------------|--------|
| wordlist 3,059 語 | ✅ | ✅ 全語 | `ipa_actual_ga` 192語 / `ipa_actual_rp` 一部 |
| connected_speech 201 句 | ✅ | ✅ | なし（phonemic のみ） |
| weak_forms 36 語 | ✅ | ✅ | strong 形も `rp_ipa_strong` |

例（`address`）:

- GA: `/ˈæˌdrɛs/`（ɛ）
- RP: `/ˈæˌdres/`（e）→ **異なる**ため現状 RP 行が出る

例（同一ケース）:

- 多くの連結句は `ipa === rp_ipa`（例: `an apple`）→ 現状 **何も表示されない**

## 5. 実装上の課題・未決定事項

### 5.1 「同一」の判定基準

候補:

| 案 | 内容 | メリット | デメリット |
|----|------|----------|------------|
| **A. phonemic 文字列一致** | `ipa === rp_ipa`（現状の hide 条件と同じ） | 実装が簡単 | `ˈæˌdrɛs` vs `ˈæˌdres` は別扱い（正しい） |
| **B. 正規化後一致** | ストレス記号除去・Unicode 正規化後に比較 | 見た目の揺れを吸収 | 過剰に同一判定するリスク |
| **C. narrow 同士** | `ipa_actual_*` も考慮 | 実音重視 | データが語によって欠ける |

**推奨:** まず **案 A（phonemic 文字列一致）** で「同一」表示。必要なら後から正規化を追加。

### 5.2 何を「反対アクセント」として見せるか

| 候補 | 説明 |
|------|------|
| **phonemic のみ** | 現行 `altIpa()` と同じ。採点・辞書と整合 |
| **narrow があれば narrow** | 主表示と揃えるが、反対側に narrow データが無い語が多い |

**推奨:** 主表示は narrow、反対行は **phonemic（`altIpa`）** のまま（SPECIFICATION.md の設計意図と一致）。

### 5.3 どの画面に出すか

| 優先度 | 画面 | 理由 |
|--------|------|------|
| 高 | **Reveal** | 既に `#rAltIpa` あり。同一表示の仕様変更だけで効果大 |
| 高 | **Decode / 連結音 出題カード** | ユーザーが添付した文脈に近い。主 IPA 下に小さく |
| 中 | **Mode B Study（意味確認後）** | 発音学習の一環として自然 |
| 低 | **Encode 出題中** | 単語→IPA が主目的のため優先度低 |
| 要検討 | **語彙ブラウザ** | 既に GA+RP 併記。同一時の「同じ」表現だけ揃える |

### 5.4 同一時の表示案（Claude 相談用ドラフト）

```
GA 運用・同一時:
  RP（英）: 同じ (/ˈædrɛs/)

GA 運用・異なる時（現状維持）:
  RP（英）: /ˈæˌdres/
```

i18n 案:

- `reveal.alt_same` … 「同じ」「Same」「相同」等

### 5.5 TTS

現状: スピーカーは **運用アクセントのみ**（プリフェッチも同様）。  
反対アクセントの音声まで要望に含めるかは **スコープ外として明示**した方がよい（工数・キャッシュが倍増）。

### 5.6 弱形・連結音の特殊性

- **弱形 Reveal:** すでに `強形 ↔ 弱形` の対比（`#rCsMeta`）。その下に GA/RP 行を足すと情報過多の可能性  
- **連結音:** フレーズ単位の `rp_ipa` は多くが GA と同一 → 「同一」表示が頻出する

## 6. Claude に相談するとよいこと（チェックリスト）

実装前に **5〜10 分の設計確認**で十分な項目:

1. **同一判定** — phonemic 文字列一致でよいか。ストレス記号の有無だけ違うケースはどうするか  
2. **同一時コピー** — 5言語で自然な短い表現（`同じ` / `Same as above` / 記号のみ 等）  
3. **表示優先順** — Reveal のみ先にやるか、出題カードまで一括か  
4. **弱形・連結音** — GA/RP 行を出すか、Reveal の単語モードだけに限定するか  
5. **語彙ブラウザ** — 現行の「RP 行を隠す」を「同じ」と表示に変えるか

**相談不要（Cursor で進めてよい）:**

- 共通関数 `renderAltAccentLine(c, elId)` の抽出  
- 各画面への DOM 追加・CSS（`.alt-ipa` 流用）  
- i18n キー追加（文言はドラフトで入れ、後から Claude 校正可）  
- `refreshRevealIpa` の `alt === phonemic` 分岐を「同一表示」に変更

## 7. Cursor 実装の推奨ステップ（参考）

1. **Phase 1 — Reveal のみ**  
   - `altIpa` あり & 同一 → `RP（英）: 同じ (/…/)`  
   - 異なる → 現状維持  
2. **Phase 2 — Decode / Mode B Study**  
   - 主 IPA 下に同じコンポーネントを配置（非活性・小さめ）  
3. **Phase 3 — 語彙ブラウザ整合**  
   - `rpLine` 非表示を「同じ」1 行に変更するか検討  

各 Phase 後に visual 確認。データ変更は不要。
